项目中bug总结

1、<p>标签不要嵌套<div>标签
eg:<p>测试一下<div>块元素</div>与内联元素的差别</dt>
当一个<p>标签还没结束时，遇到下一个块元素就会把自己结束掉，上面的例子中，div会自己产生一个新的行，浏览器会默认把它们处理成<p>标签自动结束。
2、另外还有有几个特殊的块级元素只能包含行内元素，不能再包含块级元素
Eg：h1,h2,h3,h4,h5,h6,p,dt。
3、placeholder自适应：
页面中使用<input>标签并且有属性值placeholder，在页面横屏再转回竖屏时，ios和android都会无法自适应页面。
解决方案是在<input>外层容器中加overflow:hidden
4、Android2.3的overflow问题：
在android2.3及以下系统版本的浏览器不支持overflow：scroll / auto，即在页面元素里面的内容如果超过了父元素或祖先元素的高度是无法滚动的，可以通过css3的transform来实现滚动，代表作：iscroll。
5、Android2.3会出现点击聚焦的情况出现黄色的边框.
通过对相应的元素设置css –webkit-tap-highlight-color: rgba(0,0,0,0)解决，另外一些高版本的系统也可能会出现有黄色边框的情况，这时候试着用outline：none来解决（公用样式表中有设为全局样式）
6、页面同时出现fixed和input，焦点在input上时候，安卓某些版本会由于键盘的向上推移致使fixed内容盖住原有位置内容。
解决方案，设置最小高度。
7、高度自适应问题Js框架会在html外层添加一层view的结构<div id="view"></div>。
Css需要将#view 进行样式处理达到min-height=100%;如果body和内容背景颜色不一致会影响页面美观。
8、padding值计算问题
input 、textarea 有padding属性值，高版本系统会加上padding值，需要设置box-sizing:border-box;
9、行内元素之间html空白解决
a，比如粗暴地完全删除空白;
b，目前用的较多方案是在外层元素设置font-size：0，同时在内层元素上指定具体大小;
c，html注释法，这种方式比较渣，但效果不错，使用HTML的注释标记顶替元素之间的空白。
c，其它方式暂不推荐，有好方法后续添加。
10、使用display: -webkit-box;display: flex;弹性布局时，如果内联元素需要设置宽度，ios7.0以下版本需要修改为块级元素；
正确方案： 如：display: inline-block;/display:block;
11、使用display: -webkit-box;display: flex;弹性布局时，子元素不能只设置flex: 1;目前测试发现ios7.0以下设备没有宽度支持导致间隔和文字靠右都无法正常显示，
正确的解决方案为： -webkit-box-flex: 1;flex: 1;
12、旋转某元素时，所有的transform属性必须有webkit前缀ios7以下某些设备不兼容；
解决下如： -webkit-transform: rotate(-18deg);transform: rotate(-18deg); -webkit-transform-origin:top center;transform-origin:top center;
